USABILITY TEST
介紹
Introduce
可用性測試可以幫助設計團隊確定人們在產品方面遇到的困難，通過及時反饋和建議幫助我們進行產品的改善，也可以幫助我們了解一款產品或一項功能上架前的風險。
Nielsen Norman(美國UI.UX諮詢公司)將Usability Testing的目的總結為以下三點：
1.發現設計問題
2.發現提升機會
3.理解使用者行為和偏好
01page
USABILITY TESTING
一般來說，可用性測試可以和使用者面對面進行，也可以遠端視訊進行。通常根據任務時間，錯誤率、成功率、高效性和滿意度來衡量測試結果。其中，不僅需要研究人員具有良好的溝通和洞察能力，也需要時刻保持旁觀者的態度，盡可能保持中立態度。
02page
USABILITY TESTING
步驟
Process
可用性測試分為以下7個步驟：
1.明確測試時間和目的。
2.定義測試樣本。
3.規劃測試過程、時間表、參與人員等。
4.確定測試任務，明確每個任務的測試目的。
5.準備使用者問捲和提問，注意措辭方式，確定測試過程中的記錄方式，整理快速筆記的方法。
6.進行測試。
7.後期分析使用者測試結果，整理報告。

USABILITY TESTING
評估階段
當已經有一個功能方面相對完整的原型時，為了測試使用者是否能夠輕鬆達到目標，完成任務。測試設備是功能性原型，對於UI視覺要求不高，對於功能的完善性要求很高。

USABILITY TESTING
驗證階段
當產品已經基本完工，在投入市場之前，為了測試產品質量是否達到了市場標準和使用者期待。測試設備是最終原型或已開發出的最終產品。

USABILITY TESTING
比較階段
對於同一功能，設計不同的達成途徑。這時候需要通過測試比較這些不同途徑，尋找最優的解決方式。測試設備根據測試目的不同，可以是設計稿、功能性原型或是最終產品。

USABILITY TESTING
持續提升階段
在產品持續改版的過程中，測試使用者完成任務的高效性。或是因為市場需求變化，測試使用者對於某一功能或整體產品的滿意程度。測試設備是最終面向市場的產品。

Step2-定義測試樣本
Sample
根據測試的目的和預算水平，我們的樣本數量也不同。那麼如何知道適合產品的樣本數量呢？
有數據和算力支撐的使用者體驗總是更具有說服力，MesuringU就是一個極好的量化使用者體驗的工具。
11page
USABILITY TESTING
MesuringU
這是網站上的一個樣本量計算器，基本公式是：
為了發現__%的介面問題，且為了找到在人群中的出現率是__(0到1之間)的問題，平均需要測試__個使用者。
例如：
為了發現85%的介面問題，且為了找到在人群中的出現率是0.3的問題，平均需要測試5.32個使用者。
12page
USABILITY TESTING
MesuringU
問題在人群中的出現率(Problem Occurence)的值在0到1之間，數值越小，說明出現的概率越低，越不容易發現。測試的使用者數量越多，發現出現概率低的問題的可能性就越大。
13page
USABILITY TESTING
篩選樣本
這個階段並不需要使用特別詳細的使用者形象，但為了使測試結果更具有代表性，可以利用以下條件篩選：
性別，薪資，教育程度，生活地區，語言，年齡，使用裝置，是否有使用經驗...等
如果目標測試的使用者是某一特定地區或某一特定類型的人群，那麼樣本數量和組建方式會有很大的不同。

Step3-規劃測試過程
Sample
在規劃測試過程前，我們需要明確測試的方式：是遠端測試還是與面談測試。為了不影響最終測試結果，需要保證樣本的測試方式統一。
15page
USABILITY TESTING
遠端測試：
需要首先確定使用者是否能在自己的設備上安裝測試產品，測試時是否需要其他軟體輔助錄製過程和視訊，如何徵得使用者同意和建立信任。
面談測試：
選擇一個能夠讓使用者較為放鬆的環境，可以是佈置的比較溫馨的會議室。另外如果是App測試我們需要分別進行Android版和iOS版的可用性測試。
如果測試需要私人資料(如：銀行帳戶或登入社群帳號...等)，需要準備替代資料以便使用者可無顧慮進行測試。
16page
USABILITY TESTING
一個10-20人的可用性測試，每個使用者的測試時間控制在20-45分鐘左右，流程如下：
1.測試介紹：簡單介紹測試目的和流程，盡量讓使用者漸進入放鬆狀態，取得錄音錄影和資料授權的許可，簽訂保密協議...等。
2.測試前問卷：測試產品前，填寫一個簡單的問卷，除了簡單的個人資料外需要了解使用者對相關產品使用經驗和習慣等。
3.測試場景與任務：提供任務進行測試，在使用者進行任務過程觀察使用者表情和肢體動作，在完成任務後提出一些簡單的口頭問題。
4.任務後問卷：在完成一個任務後，用ASQ(After Scenario Questionnaire)梯度問卷調查滿意度，再進行下一個任務的測試。
5.測試後問卷：在所有任務完成後，讓使用者填寫關於整體滿意度的梯度問卷，通常使用的是SUS(System Usability Scale)系統可用性量表。

Step5-準備問卷和提問
Questionnaires
盡量減少對使用者的引導。例如：避免「您喜歡嗎？」這類帶有引導答案的問題，將提問改為「您的感受是如何呢？」，將得到更多使用訊息和減少很多測試當中的認知誤差。
依流程需要準備3種類型的問卷：
22page
USABILITY TESTING
測試前問卷(5-10個問題)
傾向於選擇題，關於個人基本資料、使用經驗、頻率...等簡單問題，便於測試後歸納資料。
23page
USABILITY TESTING
任務後問卷
每完成一個場景任務後的問卷，側重於任務滿意度。
常見的有：ASQ(After-Scenario Questionnaire)、SEQ(Single Ease Question)、SMEQ(Subjective Mental Effort Questionnaire)、ER(Expectation Ratings)、UME(Usability Magnitude Estimation)。
24page
USABILITY TESTING
ASQ問卷
3個簡單的問題，如：難易度感受、完成時間感受和訊息提示感受；
給予7個梯度的選擇，如：非常不滿意 -> 不滿意 -> 有點不滿意 -> 普通 ->滿意 -> 很滿意 -> 非常滿意。
25page
USABILITY TESTING
SEQ問卷
單項難易度，一個題目，使用1~7分制評估任務的整體難易度。

Step6-進行測試
Test
測試過程中一般有兩個測試者，一個負責觀察，一個負責筆記。
觀察者：觀察使用者的表情、肢體動作。在測試中保持和使用者的交流狀態，在使用者猶豫時給予思考時間後進行詢問，但不能引導使用者的判斷。
記錄者：使用快捷的記錄方式。可以分別記錄使用者的行為、任務完成路徑和想法。

Analysis & Reporting
30page
USABILITY TESTING
基於問題的結果分析
首先我們可以通過測試過程中的觀察記錄，提取出使用者的問題和主觀感受，歸納過程中出現的問題，製作問題出現率的圖表。
例如：我們測試了6個使用者，總結出測試過程中的7大使用問題。
一個問題出現的頻率越高，代表對應的介面設計越該重視。
31page
USABILITY TESTING
基於任務的結果分析
通過前面ASQ，SEQ和SUS拿到的測試結果，整理出相關數據圖表，便於分析與報告。
例如：我們測試了6位使用者，5個任務。
除了這些數據圖表，還可以利用Excel,Airtable, Dovetail等工具產出更複雜的交叉圖表進一步進行分析。
